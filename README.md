## 项目简介 (Introduction)

本项目是一个面向**工业物联网 (IIoT) 边缘计算场景**的嵌入式网关原型。系统以 STM32F103 微控制器为核心，深度整合 FreeRTOS 实时操作系统，旨在向下打通多维度的传感器环境数据采集，向上提供标准、可靠的 Modbus RTU 从机（Slave）服务。

除了基础的协议转换与数据透明传输，本项目更侧重于**“工业级稳定性”**的底层构建。通过引入 DMA 零阻塞通信流、多任务并发下的资源竞态保护、离线参数的安全固化机制，以及高度防弹的 CLI 交互引擎，重点攻克了嵌入式设备在复杂工业现场中容易出现的丢帧、内存越界及冷启动假死等工程痛点。

## 2. 核心系统模块设计

### 2.1 Modbus RTU 从机协议栈
* **支持功能码**：深度适配 03 (读保持寄存器)、05 (写单个线圈)、06 (写单个寄存器)。
* **高性能驱动架构 (核心亮点)**：底层采用 **UART DMA + IDLE 空闲中断 + RingBuffer (环形缓冲区)** 的组合，彻底根除高频通信下的丢帧隐患。
* **零阻塞通信数据流**：
    1. **【主机下发】**：上位机（电脑/PLC）通过总线下发 Modbus 指令帧。
    2. **【物理转换】**：板载 RS485 芯片捕获差分信号，将其还原为 TTL 逻辑电平送入 STM32 串口。
    3. **【硬件搬运】**：DMA 控制器全程零 CPU 介入，将连续的数据流“无感”搬运至底层接收区。
    4. **【中断转存】**：总线空闲触发 IDLE 硬件中断，CPU 瞬间介入将整帧数据推入 RingBuffer，并释放信号量。
    5. **【异步解析】**：Modbus 独立任务被唤醒，从缓冲区提取纯净数据包完成 CRC 校验与业务处理。
* **设备控制与配置**：支持通过线圈指令远程控制板载 LED 状态；通过寄存器读写指令，实现对报警阈值的远程获取与动态篡改。
* **容错机制**：内置 CRC16 自动纠错反馈。若接收到校验和错误的报文，底层协议栈会计算并返回正确的校验和至上位机，大幅提升开发联调效率。

### 2.2 数据采集与清洗
* **传感器接入**：集成 DHT11 数字温湿度传感器与 MQ-2 烟雾传感器（结合 STM32 内部 12 位 ADC 进行模拟量采样），实现多维度环境监测。
* **数字滤波算法**：
    * **尖峰抑制**：采用中值平均滤波算法，通过 $O(N)$ 的一次性遍历完成极值剔除与加权平均，有效滤除高频电磁干扰产生的瞬时脉冲。
    * **冷启动优化**：滤波器内置初始状态捕获逻辑，避免开机首帧数据从 0 开始的惯性滞后，确保传感器读数在开机瞬间即可达到稳定输出。
* **数据封装**：传感器原始数据统一抽象为 `SensorData_t` 结构体，通过标准化接口分发，实现了驱动硬件与上层业务逻辑的完美解耦。

### 2.3 基于 EventGroup 的异常监测
* **事件驱动模型**：定义了 FreeRTOS 事件组位图（`BIT_TEMP_HIGH = 1<<0`, `BIT_SMOKE_HIGH = 1<<1`）。
* **高响应报警机制**：后台报警任务通过 `xEventGroupWaitBits()` 阻塞监听事件。当任意环境参数越限，对应事件位由 0 置 1，系统将瞬间唤醒任务并触发声光报警（蜂鸣器间歇鸣叫 + 红色 LED 闪烁），彻底摒弃了低效的全局轮询。

### 2.4 本地 UI 交互与 OLED 显示
设计了基于 0.96寸 OLED 的三级交互菜单，并通过 3 键控制体系实现人机交互：
* **Page 0 (总览页)**：实时展示测量到的当前温度与湿度。
* **Page 1 (安防页)**：显示气体浓度实时数据，以及当前安防警报的触发状态。
* **Page 2 (配置页)**：显示当前设定的报警阈值。在该页面下，按键 2 负责递增阈值，按键 3 负责递减阈值，按键 1 则负责在三个页面之间循环切换。

### 2.5 Flash 离线参数固化系统
* **掉电持久化**：无论是通过本地按键还是远程 Modbus 下发修改的报警阈值，系统均会自动触发存储程序，确保网关断电重启后配置参数不丢失。
* **安全扇区擦写**：基于 STM32 Flash "先擦除后写入"（擦除置 `0xFF`，写入拉低至 `0`）的物理特性，在系统 Flash 的尾部独立开辟了一整页（Page）专门存放阈值参数。该设计确保了擦除操作绝不会波及或损坏主程序的代码数据。

### 2.6 CLI 命令行调试引擎 (UART1)
* **状态日志打印**：通过串口 1 实时输出系统运行轨迹及按键动作日志。
* **动态指令解析 (资源可伸缩)**：受限于部分芯片的 RAM 空间，本地 CLI 解析任务 (`UARTTask`) 设计为可裁剪模块。解除代码注释并启用该任务后，网关支持极其鲁棒的字符串指令集：
    * 支持通过 `GET TEMP_HIGH` 查询对应配置（兼容 `#`、`\r` 或 `\n` 多种帧结束符）。
    * 支持通过 `SET TEMP_HIGH [数值]` 直接在终端修改阈值，并自动触发底层的 Flash 固化逻辑。

## 技术栈与设计模式
* **架构模式**：
    * **生产者-消费者模型**：基于 DMA 接收与 FreeRTOS 队列（Queue）解耦了高速数据流采集与耗时协议解析。
    * **状态机模式**：应用于底层按键防抖与多级 OLED 菜单调度。
    * **临界区保护**：应用于 Flash 安全擦写机制。
* **实时操作系统 (RTOS)**：FreeRTOS
    * 核心 IPC 机制应用：`Queue` (消息队列防丢帧)、`EventGroup` (多源报警事件同步)、`Mutex` (日志打印总线互斥)、`Software Timer` (非阻塞 UI 刷新)。
* **硬件平台**：STM32F103C8T6 (ARM Cortex-M3)。
* **底层驱动层**：STM32 HAL Library。
* **开发工具链**：STM32CubeIDE / STM32CubeMX / Git。
* ## 3. 演进计划 (Future Work)
本项目目前的架构已具备工业现场的基础通信与监测能力，为进一步提升系统的底层寿命与协议兼容性，计划在后续迭代中完成以下升级：

* **Flash 存储算法优化 (EEPROM 磨损均衡)**
    * **现状**：目前修改阈值采用的是“修改即擦除整页”的粗放策略，频繁操作会导致该页 Flash 寿命快速衰减。
    * **优化方案**：引入意法半导体（ST）官方推荐的 **EEPROM 模拟算法**。利用 Flash 只能由 `1` 写 `0`（擦除后全为 `0xFFFF`）的物理特性，采用**“顺序追加写入”**机制。新修改的阈值直接写在旧数据的物理地址之后，上电读取时由底层指针遍历寻找最后一个非 `0xFFFF` 的数据块作为有效值。仅当当前页全部写满时，才触发一次整页擦除。
    * **预期收益**：将该页 Flash 的擦写频率降低几个数量级，成百上千倍地延长芯片的使用寿命。
    
* **Modbus 协议栈全集扩展**
    * **现状**：目前协议栈精简实现了 03、05、06 功能码，满足了基本的数据读取与单点控制。
    * **优化方案**：扩充对其他标准工业功能码的解析引擎。例如增加 `01`（读线圈状态）、`02`（读离散输入，专门用于采集按键或报警引脚状态）、`04`（读输入寄存器，隔离只读的传感器实时数据区）以及 `15`/`16`（连续写多个线圈/寄存器）的支持。
    * **预期收益**：极大提升协议栈的完整度与合规性，使网关能无缝对接更复杂、高频的工业 PLC 轮询组网任务。
